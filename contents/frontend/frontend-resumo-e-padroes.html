W3C, WHATWG e os Padrões Web
  • O World Wide Web Consortium (W3C) é uma comunidade internacional onde as organizações membros, uma equipe em tempo integral e o público trabalham juntos para desenvolver os Padrões da Web. Liderada pelo inventor e diretor da Web, Tim Berners-Lee e pelo CEO Jeffrey Jaffe, a missão do W3C é levar a Web a todo o seu potencial, garantindo sempre a abertura, acessibilidade e liberdade da World Wide Web de uma perspectiva técnica;
  • Já o Web Hypertext Application Technology Working Group (WHATWG) é uma comunidade de pessoas interessadas na evolução da Web por meio de padrões e testes.
  • As Recomendações W3C definem uma Open Web Platform para o desenvolvimento de aplicativos que tem o potencial sem precedentes para permitir que os desenvolvedores criem experiências interativas ricas, alimentadas por vastos armazenamentos de dados, que estão disponíveis em qualquer dispositivo. Embora os limites da plataforma continuem a evoluir, os líderes da indústria falam quase em uníssono sobre como o HTML5 será a base para esta plataforma. Mas a força total da plataforma depende de muito mais tecnologias que o W3C e seus parceiros estão criando, incluindo CSS, SVG, WOFF, a pilha da Web Semântica, XML e uma variedade de APIs.
  • Além dos padrões para as tecnologias W3C citadas acima, o W3C possui guias e padrões para outras tecnologias Web também.
  • Esses Padrões, Recomendações, Boas Práticas, Diretrizes ou Especificações Técnicas são muito importantes para garantir que a Web seja/esteja:
    • Acessíveis para pessoas com deficiência;
    • Disponível em qualquer dispositivo, para qualquer tipo de informação, em qualquer idioma;
    • DisponívelUniversal, internacional e verdadeiramente “World Wide”;
    • Livre de royalties e baseado em padrões abertos;
    • Transformacional para a forma como os negócios são feitos; melhorando a entrega, aumentando a satisfação do usuário e reduzindo custos;
    • E cada vez mais segura.

Acessibilidade
  • Existem mais de 1 bilhão de pessoas com deficiência, ou seja, cerca de 15-20% da população;
  • Acessibilidade na Web significa que pessoas com deficiência podem usar a Web igualmente, acessando e interagindo com os conteúdos;
  • As tecnologias da Web do W3C, como HTML, CSS e outras, fornecem suporte para muitos recursos de acessibilidade;
  • Alguns exemplos de recursos de Acessibilidade na Web incluem:
    • Legendas em conteúdo de áudio e multimídia para pessoas com eficiência auditiva;
    • Layout claro e consistente para pessoas com deficiências cognitivas e de aprendizagem;
    • Suporte de teclado para pessoas que não usam o mouse, por exemplo, pessoas com deficiência física;
    • Alternativas de texto para pessoas com deficiência visual e que usam leitores de tela;
  • A Web Accessibility Initiative (WAI) provê estratégias, padrões e recursos para tornar a Web acessível a pessoas com deficiência e ela pode ser subdivida em:
    • Diretrizes de Acessiblidade:
      • Web Content Accessibility Guidelines (WCAG);
      • Authoring Tool Accessibility Guidelines (ATAG);
      • User Agent Accessibility Guidelines (UAAG).
    • Especificações Técnicas:
      • Accessible Rich Internet Applications (WAI-ARIA);
      • Áudio e Vídeo;
      • Avaliação;
      • Personalização;
      • Pronunciação

Diretrizes de Acessibilidade
  • Web Content Accessibility Guidelines (WCAG):
    • Os documentos das Web Content Accessibility Guidelines (WCAG) explicam como tornar o conteúdo da Web mais acessível para pessoas com deficiência;
    • A versão 2.0 foi aprovada no padrão ISO/IEC 40500:2012;
    • Baseados em 4 princípios:
      • Perceptível:
        • Forneça alternativas de texto para conteúdo não textual;
        • Forneça legendas e outras alternativas para multimídia;
        • Crie conteúdo que possa ser apresentado de diferentes maneiras, inclusive por meio de tecnologias assistivas, sem perder o sentido;
        • Torne mais fácil para os usuários ver e ouvir o conteúdo.
      • Operável:
        • Disponibilize todas as funcionalidades de um teclado;
        • Dê aos usuários tempo suficiente para ler e usar o conteúdo;
        • Não use conteúdo que cause convulsões ou reações físicas;
        • Ajude os usuários a navegar e encontrar conteúdo;
        • Facilite o uso de outras entradas além do teclado.
      • Compreensível:
        • Torne o texto legível e compreensível;
        • Faça o conteúdo aparecer e operar de maneiras previsíveis;
        • Ajude os usuários a evitar e corrigir erros.
      • Robusto:
        • Maximize a compatibilidade com as ferramentas de usuário atuais e futuras.
  • Authoring Tool Accessibility Guidelines (ATAG):
    • Os documentos de Authoring Tool Accessibility Guidelines (ATAG) explicam como:
      • Tornar as próprias ferramentas de autoria acessíveis, para que pessoas com deficiência possam criar conteúdo da Web;
      • Ajudar os autores a criar conteúdo da Web mais acessível, especificamente: habilitar, apoiar e promover a produção de conteúdo em conformidade com as WCAG.
    • Ferramentas de autoria são softwares e serviços que “autores” (desenvolvedores da Web, designers, escritores, etc.) usam para produzir conteúdo da Web (páginas da Web estáticas, aplicativos da Web dinâmicos, etc.);
    • Alguns exemplos de ferramentas de autoria:
      • Ferramentas de criação de páginas da Web, por exemplo, editores de HTML what-you-see-is-what-you-get (WYSIWYG);
      • Sistemas de Gerenciamento de Conteúdo (CMS);
      • Sistemas de Gerenciamento de Aprendizagem (LMS);
      • Sites que permitem aos usuários adicionar conteúdo, como blogs, wikis, entre outros;
      • Sites de compartilhamento de fotos, fóruns online e sites de redes sociais;
      • Entre outros.
  • User Agent Accessibility Guidelines (UAAG):
    • Os documentos das Diretrizes de Acessibilidade do Agente do Usuário (UAAG) explicam como tornar os agentes do usuário acessíveis para pessoas com deficiências;
    • Alguns exemplos de agentes de usuário:
      • Navegadores;
      • Extensões de navegador;
      • Reprodutores de mídia;
      • Leitores;
      • E outros aplicativos que processam conteúdo da Web.
    • UAAG é principalmente para desenvolvedores destes agentes de usuário;
    • Algumas necessidades de acessibilidade são mais bem atendidas no navegador do que no conteúdo da Web, como personalização de texto, preferências e acessibilidade da interface do usuário.

Especificações Técnicas
  • Accessible Rich Internet Applications (WAI-ARIA):
    • O ARIA fornece semântica para que os autores possam transmitir os comportamentos da interface do usuário e informações estruturais para tecnologias assistivas (como leitores de tela);
    • Sua especificação fornece uma ontologia de funções, estados e propriedades que definem elementos de interface de usuário acessíveis;
    • O conjunto ARIA inclui especificações de mapeamento de API que fornecem orientação de implementação de agente de usuário, além de módulos para Gráficos e Publicação Digital.
  • Áudio e Vídeo:
    • O Web Video Text Tracks Format (WebVTT) é um formato para legendas, descrições de vídeo de texto e outros metadados alinhados com o conteúdo de áudio ou vídeo;
    • O Timed Text Markup Language (TTML) destina-se a ser usado para transcodificação ou troca de informações de texto cronometrado entre formatos de conteúdo de distribuição legado para subtítulos e legendagem.
  • Avaliação:
    • A Visão Geral do Teste de Conformidade de Acessibilidade (ACT) estabelece e documenta regras para testar a conformidade do conteúdo da web com os padrões de acessibilidade;
    • A Avaliação e Visão Geral da Linguagem de Relatório (EARL) é um formato legível por máquina para expressar os resultados do teste;
    • A Visão Geral da WCAG-EM: Metodologia de Avaliação de Conformidade de Acessibilidade de Site é uma abordagem para determinar o quão bem um site está em conformidade com as Diretrizes de Acessibilidade de Conteúdo da Web (WCAG).

Internacionalização (i18n)
  • 'Internacionalização' às vezes é abreviado para 'i18n' em inglês, porque há 18 caracteres entre o 'i' e o 'n';
  • Internacionalização é um padrão que recomenda que todo conteúdo ou aplicativo deve ser projetado e desenvolvido com recursos que permitam seu uso em diferentes idiomas, culturas, sistemas de escrita, scripts, etc.;

Unicode

  • Unicode é um conjunto de caracteres universal, ou seja, um padrão que define, em um só lugar, todos os caracteres necessários para escrever linguagens em uso em computadores;
  • É um superconjunto de todos os outros conjuntos de caracteres que foram codificados;
  • Conjunto de caracteres é diferente de uma codificação de caracteres;
  • Hoje em dia, você deve sempre escolher a codificação de caracteres UTF-8 para seu conteúdo ou dados. Essa codificação Unicode é uma boa escolha porque você pode usar uma única codificação para lidar com qualquer caractere que provavelmente encontrará, o que simplifica muito as coisas;
