O W3C e os Padrões da Web

As Recomendações W3C definem uma Open Web Platform para o desenvolvimento de aplicativos que tem o potencial sem precedentes para permitir que os desenvolvedores criem experiências interativas ricas, alimentadas por vastos armazenamentos de dados, que estão disponíveis em qualquer dispositivo. Embora os limites da plataforma continuem a evoluir, os líderes da indústria falam quase em uníssono sobre como o HTML5 será a base para esta plataforma. Mas a força total da plataforma depende de muito mais tecnologias que o W3C e seus parceiros estão criando, incluindo CSS, SVG, WOFF, a pilha da Web Semântica, XML e uma variedade de APIs.

Além dos padrões para as tecnologias W3C citadas acima, o W3C possui guias e padrões para outras tecnologias também.

Esses Padrões, Diretrizes, Recomendações, Boas Práticas ou Especificações Técnicas são muito importantes para garantir que a web seja/esteja:

  • Acessíveis para pessoas com deficiência;
  • Disponível em qualquer dispositivo, para qualquer tipo de informação, em qualquer idioma;
  • Universal, internacional e verdadeiramente “World Wide”;
  • Livre de royalties e baseado em padrões abertos;
  • Transformacional para a forma como os negócios são feitos; melhorando a entrega, aumentando a satisfação do usuário e reduzindo custos;
  • E cada vez mais segura.


Acessibilidade

  • Mais de um bilhão de pessoas possui algum tipo de deficiência, ou seja, cerca de 15-20% da população;
  • A Acessibilidade na Web significa que essas pessoas com deficiência podem usar a web igualmente, acessando e interagindo com os conteúdos;
  • As tecnologias da Web do W3C, como HTML, CSS e outras, fornecem suporte para muitos recursos de acessibilidade;
  • Alguns exemplos de recursos de acessibilidade da Web incluem:
    • Legendas em conteúdo de áudio e multimídia para pessoas com deficiência auditiva;
    • Layout claro e consistente para pessoas com deficiências cognitivas e de aprendizagem;
    • Suporte de teclado para pessoas que não usam o mouse, por exemplo, pessoas com deficiência física;
    • Alternativas de texto para pessoas com deficiência visual e que usam leitores de tela;
  • A Iniciativa de Acessibilidade da Web (WAI) provê estratégias, padrões e recursos para tornar a web acessível a pessoas com deficiência;
  • A WAI pode ser divida nas Diretrizes de Acessibilidade e nas Especificações Técnicas;

Especificações Técnicas

  • Accessible Rich Internet Applications (WAI-ARIA):
    • O ARIA fornece semântica para que os autores possam transmitir os comportamentos da interface do usuário e informações estruturais para tecnologias assistivas (como leitores de tela);
    • A especificação ARIA fornece uma ontologia de funções, estados e propriedades que definem elementos de interface de usuário acessíveis;
    • O conjunto ARIA inclui especificações de mapeamento de API que fornecem orientação de implementação de agente de usuário, além de módulos para Gráficos e Publicação Digital.
  • Áudio e Vídeo:
    • O Formato de Trilhas de Texto de Vídeo da Web (WebVTT) é um formato para legendas, descrições de vídeo de texto e outros metadados alinhados com o conteúdo de áudio ou vídeo;
    • O Timed Text Markup Language (TTML) destina-se a ser usado para transcodificação ou troca de informações de texto cronometrado entre formatos de conteúdo de distribuição legado para subtítulos e legendagem.
  • Avaliação:
    • A Visão Geral do Teste de Conformidade de Acessibilidade (ACT) estabelece e documenta regras para testar a conformidade do conteúdo da web com os padrões de acessibilidade;
    • A Avaliação e Visão Geral da Linguagem de Relatório (EARL) é um formato legível por máquina para expressar os resultados do teste;
    • A Visão Geral da WCAG-EM: Metodologia de Avaliação de Conformidade de Acessibilidade de Site é uma abordagem para determinar o quão bem um site está em conformidade com as Diretrizes de Acessibilidade de Conteúdo da Web (WCAG).
