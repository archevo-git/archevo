HTML5: Abreviação de HyperText Martkup Language ou Linguagem de Marcação de Hipertexto.

Descrição: Linguagem padrão para estruturar a publicação de conteúdos (áudios, vídeos, imagens, textos, etc.) em páginas Web, através do uso de tags. Foi criado para ser independente de plataformas (ou Cross Platform), browsers e outros meios de acesso, já que interoperabilidade significa menos custo: um único código HTML pode ser lido em diversos dispositivos.

Criação: 1991, por Tim Berners-Lee

Extensão do Arquivo: .html, .htm

MIME: text/html

Tipo de Formato: Linguagem de marcação

Padrões, Recomendações e Boas Práticas: World Wide Web Consortium (W3C) e WHATWG

Padrões, Recomendações & Boas Práticas

  • O padrão HTML não requer tags em minúsculas, mas o W3C recomenda minúsculas em HTML e exige minúsculas para tipos de documentos mais restritos, como XHTML;
  • O padrão HTML não requer nomes de atributos em letras minúsculas, mas o W3C recomenda atributos em minúsculas em HTML e exige atributos em minúsculas para tipos de documentos mais restritos, como XHTML;
  • O W3C recomenda o uso de aspas ("") em torno dos valores dos atributos HTML e exige seu uso para tipos de documentos mais restritos, como XHTML;
  • HTML permite espaços em torno de sinais de igual. Mas menos espaço é mais fácil de ler e agrupa melhor as entidades;
  • Se usar um editor como o VSCode, vá na opção "Ver" e ative a opção "Quebra De Texto" para que não seja necessário rolar para a esquerda e direita para visualizar a linha de código;
  • Não adicione linhas em branco, espaços ou recuos sem um motivo;
  • Para facilitar a leitura, adicione linhas em branco para separar blocos de código grandes ou lógicos;
  • Para facilitar a leitura, adicione dois espaços de recuo. Não use a tecla tab.
  • No HTML não é obrigatório fechar elementos que não tem a tag final, como o <br>. Agora, se você espera que um software XML ou XHTML acesse sua página, mantenha a barra de fechamento ("/"), porque ela é necessária em XML e XHTML.
  • Para nomes de arquivos, além de não deixar espaços, é recomendado sempre usar letras minúsculas para evitar erros em alguns web servers, como os Apache e Unix, ou após uma eventual migração para eles.

Padrões, Recomendações & Boas Práticas

  • <header>:
    • Pode conter ou não um <hgroup>;
    • Não é dividido por <section>;
    • Podem ezistir vários elementos <header> em um documento HTML. No entanto, <header> não pode ser colocado em um <footer>, <address> ou outro elemento <header>.
  • <nav>:
    • Pode estar dentro ou fora do <header>;
    • Nem todos os grupos de links em uma página precisam estar em um elemento dele, já que se destina principalmente a seções que consistem em blocos de navegação principais;
    • Links para várias páginas de um site, como os termos de serviço, a página inicial e uma página de direitos autorais, podem estar contidos no <footer>;
    • Dentro de um mesmo documento, podem existir um <nav> para a navegação primária e outro(s) para as navegações secundário(s);
    • Navegadores, como leitores de tela para usuários com deficiência, podem usar esse elemento para determinar se a renderização inicial desse conteúdo deve ser omitida.
  • <main>:
    • Representa o conteúdo dominante da página;
    • Um documento não deve ter mais de um elemento <main> que não tenha o atributo hiddenespecificado.
    • Não pode ser descente das tags <header>, <nav>, <aside>, <article> ou <footer>, mas pode ser descendente das tags <section>;
  • <article>:
    • Deve fazer sentido por si só e deve ser possível distribuí-lo independentemente do resto do site;
    • Pode ser dividido em uma ou mais <section> que, por sua vez, pode(m) conter outros <article>;
    • Pode estar aninhado dentro de outro <article>, porém, as informações do autor (<address>) não se aplicam a elementos do <article> aninhado;
    • Pode conter seus próprios <header> e <footer>.
  • <section>:
    • Não precisa necessariamente estar dentro de um <article>, na verdade pode até existir sem ele;
    • Geralmente tem seu próprio título (<h1>..<h6>);
    • Pode conter <article>;
    • Os autores são encorajados a usar o elemento <article> em vez do elemento <section> quando faz sentido organizar o conteúdo do elemento;
    • Não é um elemento de contêiner genérico. Quando um elemento é necessário apenas para fins de estilo ou como uma conveniência para scripts, os autores são incentivados a usar o elemento <div>. Uma regra geral é que o elemento <section> é apropriado apenas se o conteúdo do elemento for listado explicitamente no esboço do documento.
    • Pode ter outra <section> aninhada.
  • <aside>:
    • Representa uma seção de uma página que consiste em um conteúdo tangencialmente relacionado ao conteúdo principal, mas que pode ser considerado separado desse conteúdo;
    • Pode ter seu(s) próprio(s) <nav>;
    • Um documento pode conter mais de um <aside>.
  • <footer>:
    • Pode ser o rodapé da página inteira ou de alguma seção dela, como um <article> ou uma <section>;
    • Pode conter ou não um <address>;
    • Um documento pode conter mais de um <footer>;
    • Pode estar logo no começo do <body>.
  • <address>:
    • Pode estar dentro ou fora do <header>;
    • Não deve ser usado para representar endereços arbitrários (por exemplo, endereços postais), a menos que esses endereços sejam de fato as informações de contato relevantes. O elemento <p> é o elemento apropriado para marcar endereços postais em geral;
    • Não deve conter outras informações além de informações de contato;
    • Geralmente faz parte do <footer>.

Principais Elementos e Atributos do Cabeçalho
  • <!DOCTYPE html>: Primeiro elemento, definido na primeira linha, antes do <head> e que declara que o documento está escrito em HTML5;
  • lang: Atributo da tag <html>, definida antes do <head> e que especifica o idioma que o documento irá usar;
  • <meta>: Contém meta informações sobre o documento;
  • <base>: Especifica a URL base e/ou destino para todas as URLs relativos em um documento;
  • charset: Atributo de <meta> que define o conjunto de caracteres usado no documento;
  • <title>: Especifica um título para o documento;
  • <style>: Define estilos CSS usados no documento;
  • <link>: Permite que os autores vinculem seus documentos a outros recursos, como um ".css" com a especificação dos estilos CSS;
  • <script>: Define um script para executar funções na página, a partir de comandos embutidos ou definidos em um arquivo ".js" remoto, por exemplo.

Padrões, Recomendações & Boas Práticas

  • Sempre declare o tipo de documento (com )<!DOCTYPE html>);
  • Sempre inclua o atributo lang dentro da tag <html>, para declarar o idioma da página. Isso serve para auxiliar os mecanismos de pesquisa e navegadores;
  • Para garantir a interpretação adequada e a indexação correta do mecanismo de pesquisa, o idioma e a codificação de caracteres, através do atributo charset, devem ser definidos o mais cedo possível em um documento HTML;
  • Inclua o seguinte elemento <meta> em todas as suas páginas para fornecer ao navegador instruções sobre como controlar as dimensões e a escala da página, garantindo maior responsividade:
  • O <title> é muito importante para a otimização do mecanismo de pesquisa (SEO). O título da página é usado pelos algoritmos do mecanismo de pesquisa para decidir a ordem ao listar as páginas nos resultados da pesquisa. Portanto, tente tornar o título o mais preciso e significativo possível;
  • Use uma sintaxe simples no <link> para vincular o CSS (o atributo type não é necessário);
  • Use uma sintaxe simples para carregar scripts externos com o elemento <script>(o atributo type não é necessário);

  • URL Relativa:

    • Se a URL começar sem a barra, ela será relativa à página atual:
      src="img_girl.jpg".
    • Se a URL começar com uma barra, ela será relativa ao domínio:
      src="/images/img_girl.jpg".

Padrões, Recomendações & Boas Práticas

  • Utilize URLs Relativas para os conteúdos e recursos da sua página. Assim, caso o diretório do domínio seja alterado, não ocasionará em links quebrados;
  • A tag <base> especifica a URL base e/ou destino para todos as URL Relativas em um documento;
  • Imagens externas podem estar protegidas por direitos autorais. Se você não obtiver permissão para usá-las, pode estar violando as leis de direitos autorais. Além disso, você não pode controlar imagens externas; elas podem ser removidas ou repentinamente. Assim, a recomendação é para priorizar imagens locais;
  • Sempre forneça o atributo alt para as imagens. Assim, quando as imagens não puderem ser carregadas na página, um leitor de tela poderá "ler" seu conteúdo para pessoas com deficiência visual ou com dificuldades de aprendizagem. Um leitor de tela é um programa de software que lê o código HTML e permite ao usuário "ouvir" o conteúdo exibido na tela;
  • Sempre especifique o tamanho de uma imagem. Se largura e altura não forem especificadas, a página da web pode piscar enquanto a imagem carrega;
  • Quando o width for definido como 100%, a imagem se tornará responsiva e será dimensionada para cima e para baixo ao redimensionar a janela do navegador;
  • Se a propriedade max-width for definida como 100%, a imagem será reduzida se necessário, mas nunca aumentará para ser maior do que seu tamanho original;
  • Usar o atributo style, ao invés de height e width para definir o tamanho das imagens, impede que as folhas de estilo alterem o tamanho delas;
  • Carregar imagens grandes leva tempo e pode tornar sua página da Web lenta. Prefira imagens menores ou com formatos mais leves.
  • Sempre especifique um elemento <img> como o último elemento filho do elemento <picture>. O elemento <img> é usado por navegadores que não suportam o elemento <picture> ou se nenhuma das tags <source> corresponder.

Padrões, Recomendações & Boas Práticas

  • Use <b> para textos em negrito sem nenhuma importância adicional. É uma tag não semântica e, de acordo com a especificação HTML5, deve ser usada como o último recurso, quando nenhuma outra tag é mais apropriada;
  • Use <strong> para textos em negrito com grande importância adicional. É uma tag semântica, muito útil para garantir Acessibilidade;
  • Use <i> para textos em itálico que deverão ser pronunciados com uma voz ou humor alternativo. Frequentemente usado para indicar um termo técnico, uma frase de outro idioma, um pensamento, um nome de navio, etc. É uma tag não semântica, usada apenas quando nenhuma outra tag semântica atender a necessidade;
  • Use <em> para textos em itálico e enfatizados. Um leitor de tela pronunciará as palavras com ênfase, usando ênfase verbal. É uma tag semântica, muito útil para garantir Acessibilidade;
  • Não use <s> para definir um texto que foi deletado de um documento;
  • Evite usar o elemento <u> onde pode ser confundido com um hiperlink.
  • A tag <caption> deve ser inserida logo após a tag <table>.
  • A tag <colgroup> deve ser filha de um elemento <table> e deve ser colocada antes de quaisquer outros elementos da tabela, como <thead>, <tr>, <td> etc., mas após o elemento <caption>, se presente.

Padrões, Recomendações & Boas Práticas

  • Não use o atributo placeholder no lugar de um elemento <label. Os propósitos de cada um são diferentes: <label descreve o papel do elemento do formulário, isto é, ele indica que tipo de informação é esperada. Já o atributo placeholder é uma dica sobre o formato que o conteúdo deveria ter. Há casos em que o atributo placeholder nunca é exibido para o usuário, portanto o formulário deve ser inteligível sem ele;
  • A largura padrão de um <input é de 20 caracteres. É recomendado atentar-se à essa largura para que os dados fiquem visíveis durante sua inserção.
